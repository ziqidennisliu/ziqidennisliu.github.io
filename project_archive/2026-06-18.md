## 任务摘要

优化 `GALLERY` 页面图片加载体验：先快速显示等尺寸模糊缩略图，再在图片进入视口后加载并替换为完整尺寸图片，减少图片加载导致的重新排版。

## 修改文件

- 修改 `_pages/gallery.md`
- 修改 `_sass/_base.scss`
- 更新 `project_archive/2026-06-18.md`

## 改动说明

- 将 gallery 页面中展示的图片初始 `src` 改为 Jekyll/ImageMagick 生成的 `-480.webp` 小尺寸 WebP 缩略图。
- 为每张 gallery 图片增加 `data-full-src`，保存原始 JPG 地址，用于后续高清替换。
- 为每张 gallery 图片增加真实 `width` 与 `height` 属性，使页面脚本在图片下载前即可计算等比例 justified row 布局。
- 增加 `loading="lazy"` 与 `decoding="async"`，降低初始页面加载压力。
- 更新 gallery 内联脚本：
  - 优先使用 HTML `width` / `height` 计算布局，不再等待 `naturalWidth`。
  - 页面 ready 后立即为每一行设置稳定宽高。
  - 使用 `IntersectionObserver` 在图片接近视口时加载完整 JPG。
  - 高清图加载完成后替换缩略图并移除模糊效果。
- 更新 gallery CSS：
  - 为缩略图阶段增加灰色背景。
  - 保留模糊渐变效果，并在高清图完成后平滑锐化。
